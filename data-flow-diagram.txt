Data Flow Summary:
=================

1. INGESTION: Device → MQTT → Spring → Processing
2. STORAGE: Processing → MongoDB (persistent) + Redis (cache)
3. DELIVERY: Storage → API/WebSocket → Client
4. AUTOMATION: Rules → Monitoring → MQTT Commands → Device

Key Data Types:
==============

- Telemetry Data: Temperature, humidity, sensor readings
- Device Metadata: Configuration, status, capabilities
- Automation Rules: Conditions, actions, triggers
- User Data: Authentication, preferences, dashboards
- Time Series: Historical data with timestamps
- Cache Data: Frequently accessed computed results

Performance Characteristics:
==========================

- MQTT: Low-latency, high-throughput device communication
- WebSocket: Real-time bidirectional client updates
- MongoDB: Flexible document storage, aggregation queries
- Redis: Sub-millisecond cache access, TTL expiration
- REST API: Standard HTTP with JSON payloads
